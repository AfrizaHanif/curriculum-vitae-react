Ringkasan Proyek

Deskripsi

Membangun aplikasi berbasis web untuk membantu proses penilaian dan pemilihan karyawan terbaik di BPS Provinsi Jawa Timur dengan menggunakan metode Simple Additive Weighting (SAW). Aplikasi ini dirancang untuk mempermudah pengambilan keputusan berbasis kriteria yang telah ditentukan, seperti kehadiran, capaian kinerja pegawai, dan tujuh indikator BerAkhlak.

Peran & Tanggung Jawab

Saya mengerjakan proyek ini sebagai Full-Stack Developer dan memiliki tanggung jawab yang harus saya lakukan pada proyek ini:

  1. Mengerjakan proyek sesuai dengan kebutuhan dari pengguna (Kepegawaian, Kepala BPS Jawa Timur, dan seluruh pegawai BPS Jawa Timur)
  2. Mengerjakan proyek tanpa mengubah data nilai asli yang dimiliki oleh BPS Jawa Timur

Permasalahan

Selama mengerjakan proyek ini, saya mengidentifikasi beberapa kendala / masalah yang ada. Di antaranya adalah:

  1. Presensi saat melakukan apel Senin dan senam Jumat masih menggunakan presensi manual sehingga proses rekap akan berlangsung lama karena diselingi aktivitas lain dan perlu dilakukan proses entry sampai satu minggu
  2. Belum ada pembobotan untuk menilai secara kualitatif kinerja karyawan yang didasarkan pada indikator BerAkhlak

Tujuan dan Manfaat

Saya memiliki tujuan dilakukannya proyek ini. Tujuannya adalah:

  1. Menghasilkan aplikasi penentuan karyawan terbaik pada BPS Jawa Timur
  2. Penilaian kinerja dapat ditentukan berdasarkan kinerja karyawan yang sebenarnya
  3. Dapat memenuhi kriteria kedisiplinan (Kehadiran dan Keterlambatan), kinerja karyawan (CKP) dan 7 perilaku BerAkhlak (Berorientasi Pelayanan, Akuntabel, Kompeten, Harmonis, Loyal, Adaptif, dan Kolaboratif) serta memiliki integritas yang tinggi secara objektif

Terdapat juga manfaat yang diperoleh setelah saya melakukan proyek ini:

  1. Mempermudah dan mempercepat pengguna untuk melakukan pengisian nilai kinerja untuk melakukan proses karyawan terbaik
  2. Mempermudah pengguna untuk mengakses daftar nilai di setiap kriteria

Proses & Solusi

Pada masalah-masalah yang ada, saya telah menemukan proses yang telah saya lakukan serta solusi yang telah saya temukan

Proses
  1. Setelah dilakukannya observasi dan wawancara, saya melakukan pengumpulan data mengenai masalah-masalah yang ada dan kebutuhan data untuk pembuatan aplikasi pada proyek ini
  2. Agar proyek ini berjalan dengan baik, saya menggunakan model pengembangan aplikasi Waterfall milik Sommerville tahun 2016
  3. Sebelum dilakukannya pembuatan aplikasi, saya membuat sebuah diagram (Context Diagram, DFD Level 0, dan PDM) yang memudahkan saya untuk merencanakan pembangunan aplikasi pada proyek ini
  4. Saya memilih Framework Laravel karena fitur Eloquent ORM yang memudahkan pengelolaan database relasional yang kompleks serta sistem keamanan built-in yang kuat untuk melindungi data instansi
  5. Proses pengembangan dilakukan secara bertahap, dimulai dari pembangunan skema database hingga perancangan kode pada sisi Frontend dan Backend
Solusi
  1. Menggantikan cara tradisional menjadi aplikasi web
    Saya membuat aplikasi web untuk menggantikan cara tradisional dari masalah yang ada agar memudahkan pengguna untuk mengisi data dengan cepat menggunakan fitur import file ke aplikasi web
  2. Menggunakan Simple Additive Weighting (SAW) sebagai metode pemilihan keputusan untuk perhitungan karyawan terbaik
    Untuk mementukan pemilihan karyawan terbaik menggunakan bobot, saya menggunakan Simple Additive Weighting (SAW) sebagai metode pemilihan keputusan

Tantangan & Pembelajaran

Dari proyek yang telah saya kerjakan, terdapat tantangan yang saya alami dan pembelajaran yang saya dapatkan:

Tantangan
  1. Pada proyek ini membutuhkan waktu lama untuk mempelajari metode SAW untuk digunakan pada aplikasi ini
Pembelajaran
  1. Dari proyek yang saya kerjakan, untuk ke depannya saya akan mempelajari cara mempersingkat kode program agar terlihat lebih ringkas

Hasil Akhir

Pada proyek ini telah menghasilkan hasil akhir yang telah saya kerjakan. Terdapat poin-poin hasil akhir yang ada:

  1. Kepegawaian dapat mengelola data karyawan, pengguna, kriteria, dan periode dalam satu halaman saja
  2. Dengan adanya aplikasi yang telah saya buat, pengguna dapat melakukan pemasukan data karyawan dan data nilai menggunakan impor untuk mempercepat proses saat melakukan pemasukan kedua data tersebut
  3. Proyek ini dapat melakukan analisis SAW dan menghasilkan laporan (Lap. Karyawan, Lap. Hasil Analisis SAW, Lap. Nilai Akhir, Lap. Nilai Pegawai, Lap. Karyawan Terbaik, Sertifikat Karyawan Terbaik)
  4. Total waktu responsif pada aplikasi yang telah saya buat sekitar 01.79 Menit dibandingkan dengan waktu sebelum adanya aplikasi tersebut sekitar 4 sampai 5 minggu
